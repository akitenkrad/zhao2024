# ユースケース

Zhao et al. (2024)「CompeteAI」再現実装でできること．

## 1. 仮想タウン基本実験の再現

2 レストランが 50 顧客を 15 日間奪い合う様子を実行し，マタイ効果 / 市場集中・勝者総取りが創発するか観察する:

```bash
export OLLAMA_MODEL=llama3.2:latest
cargo run --release -- run --n-firms 2 --n-customers 50 --customer-mode individual --days 15 --runs 9 --seed 42
uv run competeai-tools visualize
```

`run` の全フラグと LLM 環境変数は [CLI](cli.ja.md)，4 つの出力図は [可視化](visualization.ja.md) を参照．

## 2. 個人客 vs グループ客の比較

論文ではグルーピングが正のフィードバックループを攪乱し勝者総取りを緩和する (66.7% → 16.7%)．顧客モードを切り替える:

```bash
cargo run --release -- run --customer-mode group --group-size 4 --days 15 --runs 6 --seed 42
```

## 3. 感度分析スイープ

店舗数 × 顧客数 を走査し，マタイ効果が創発する条件を体系的に明らかにする:

```bash
cargo run --release -- sweep --n-firms-values 2,3,4 --n-customers-min 20 --n-customers-max 80 --n-customers-step 20 --days 15 --runs 5 --seed 42
uv run competeai-tools visualize-sweep
```

## 4. オフライン実行 (ライブ LLM 不要)

スクリプト化した mock クライアントで，日次ループ・出力ライタ・Python 可視化を LLM なしに検証する:

```bash
cargo run --release --example mock_smoke -- results
uv run competeai-tools visualize
```

## 5. 実行設定と LLM メタデータの確認

```bash
uv run competeai-tools show-experiment-settings --results-dir results/latest
```

## 決定論の注記

socsim コアは seed が決まれば bit 再現する．LLM レイヤはプロンプト→応答キャッシュ (`temperature=0`・固定 seed) で擬似決定論化する．ウォームキャッシュでの再実行は同一応答を再生する．ローカルモデルは論文の `gpt-4` と異なるため，再現目標は定性的である — [アーキテクチャ](architecture.ja.md) を参照．
